REQUISITOS PREVIOS
------------------
- Python 3.10-3.12
- Git instalado
- ffmpeg en PATH (para Whisper)
- Tesseract OCR instalado en C:\Program Files\Tesseract-OCR\
- Edge WebDriver compatible con tu versión de Edge


PASO 1 — Clonar OmniParser
---------------------------
Ejecutar desde C:\Users\Oscar\Desktop\Nueva_prueba\

    git clone https://github.com/microsoft/OmniParser OmniParser

Resultado esperado: carpeta OmniParser\ con util\utils.py dentro.


PASO 2 — Descargar pesos del modelo
-------------------------------------
OmniParser usa dos modelos: detección de íconos (YOLO) y descripción
de íconos (Florence-2 fine-tuned). Se descargan con la CLI de HuggingFace.

IMPORTANTE: usar "hf download" NO "huggingface-cli download"
(huggingface-cli está obsoleto y ya no funciona).

    cd C:\Users\Oscar\Desktop\Nueva_prueba\OmniParser

    hf download microsoft/OmniParser-v2.0 icon_detect/model.pt          --local-dir weights
    hf download microsoft/OmniParser-v2.0 icon_detect/model.yaml        --local-dir weights
    hf download microsoft/OmniParser-v2.0 icon_detect/train_args.yaml   --local-dir weights
    hf download microsoft/OmniParser-v2.0 icon_caption/config.json      --local-dir weights
    hf download microsoft/OmniParser-v2.0 icon_caption/generation_config.json --local-dir weights
    hf download microsoft/OmniParser-v2.0 icon_caption/model.safetensors --local-dir weights

Renombrar la carpeta de pesos del caption model:
    cd weights
    ren icon_caption icon_caption_florence

    (IMPORTANTE: el nuevo nombre NO lleva "weights\" delante,
     solo el nombre nuevo: icon_caption_florence)

Estructura final esperada:
    OmniParser\
    └── weights\
        ├── icon_detect\
        │   ├── model.pt
        │   ├── model.yaml
        │   └── train_args.yaml
        └── icon_caption_florence\
            ├── config.json
            ├── generation_config.json
            └── model.safetensors

PASO 6 — Correr el pipeline
-----------------------------
    cd C:\Users\Oscar\Desktop\Nueva_prueba\solo_nube
    python "4-Guarda_mjs_recep/procesar_respuestas.py"

Escanear QR en Edge cuando aparezca la ventana, luego presionar ENTER.

Qué hace con los audios:
  1. Busca botones "Reproducir mensaje de voz" en el DOM (aria-label).
  2. Si los encuentra: clica por coordenadas (document.elementFromPoint).
  3. Si no los encuentra: lanza OmniParser V2 con captura al 50% (más rápido).
  4. El hook de decodeAudioData captura el audio cuando WhatsApp lo decodifica.
  5. El archivo se guarda en media_whatsapp\<telefono>\ como .ogg (u otro formato).
  6. Whisper transcribe el audio a texto y lo sube a Azure junto con el resto.

Archivos guardados:
  - Imágenes: media_whatsapp\<telefono>\<timestamp>_<indice>.jpg
  - Audios:   media_whatsapp\<telefono>\<timestamp>_audio_<indice>.ogg
  - Videos:   media_whatsapp\<telefono>\<timestamp>_<indice>.mp4


NOTA SOBRE VELOCIDAD
---------------------
OmniParser (Florence-2) corre en CPU si no hay GPU NVIDIA con CUDA.
En CPU tarda ~3-10 segundos por captura de pantalla.
El código detecta CUDA automáticamente (config.py: _detectar_device()).
Si tu PC tiene GPU NVIDIA, instalar torch con CUDA acelera esto enormemente:
    pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
